Think Like a Programmer

Pretend you are writing a plugin that automatically harmonizes with MIDI input. What instructions would you give a computer to do this programmatically?
I would first define harmony as a property, then define the event as the act of harmonizing to the incoming midi input, then give the method of specific harmonization, such as "major3rd"
Pick an instrument and pretend you are writing an app that mimics that instrument. How would you model the istrument in a way the computer would understand? Think about what different objects (e.g. "string" for a guitar) it might need and what properties (e.g. "pitch"), events (e.g. "pluck"), and methods (e.g. "playPitch()") would be useful.
I'll pick the saxophone since that is my primary instrument, though I may be shooting myself in the foot since there are many variables. The primary objects would be the different buttons to press down which result in different notes. I would call these "button" and label them based on their corresonding name or finger on the sax (eg. button_lp;left pointer finger, or button_rm; right middle finger ) There are a few properties, most notably pitch, which would be determined by which objects (or buttons) are activated. Other properties include articulation, vibrato, and dynamics. An event would be blowing, in which all or some properties are activated. And the method would be combine these previously listed factors
